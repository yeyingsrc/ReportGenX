运行时高级设置

通常保持默认即可。默认配置已经兼顾稳定性与兼容性；只有在排查模板兼容问题、灰度试运行新模式时，才需要手动调整。

推荐 大多数用户直接保持：运行模式 = 自动兼容模式、隔离回退模式 = 自动兼容兜底、隔离子进程策略 = 自动兼容策略。
如果你不清楚该怎么选，就保持“自动兼容模式”。它会优先使用新执行方式，同时尽量避免因为兼容性问题直接报错失败。
只有在你启用了“独立进程试运行”时，这个配置才会明显生效。一般保留“自动兼容兜底”更稳妥。
这个选项主要给隔离执行时用。除非你正在排查某个模板在隔离子进程中的特殊问题，否则不要改。
默认 120 秒。只有模板生成特别慢时才需要调大；太小可能导致正常任务被误判为超时。
0 表示默认不启用隔离试运行。建议按 5% → 25% → 50% → 100% 逐步放量，不要一开始就全量。
默认 50，表示每执行约 50 次记录一次指标。一般不需要改，数值越小日志/统计越频繁。
只有某个模板在新模式下不稳定时再填。留空表示所有模板都按默认策略运行。
只让指定模板参与 isolated 试运行。适合先挑少量模板做验证，普通使用建议留空。
当某些模板确认不适合 isolated 时再填写。留空表示不额外禁用。
用于给单个模板单独指定试运行比例，例如 {"Attack_Defense": 100}。不确定怎么写就留空。
变更提示： 如果你只是正常使用软件，建议保持默认值不动；只有在模板兼容排查或灰度试运行时，再按 5% → 25% → 50% → 100% 逐步调整。
